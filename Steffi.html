Steffi's Journal

January February March April May

Week 1: 24/01/2020

Start of Semester

The parts required to build the automated hydroponices system were being discussed and approved with our Project Supervisor. We settled on using 3 sensors in the project, ph, turbidity and soil moisture, as we found these were sufficient to monitor the important elements of the hydroponic system. 2 of these sensors are analog therefore we would need to purchase Arduino to take readings from these sensors. I have no experience with the Arduino board, this will be interesting. We also decided to borrow a Raspberry Pi 3 from the department instead of purchasing one to reduce cost.

Week 2: 31/01/2020

Problems with ordering parts

Zane got into some issues when attempting to order the parts for the project. We are currently trying to determine what is the likely cause of the problem to fix it. I researched similar hydroponic projects utilizing the Pi and how they implemented their system to help get some direction for ours. I also created the project folder in my drive which will house all important documents and items related to the project that the Project supervisor has access to. We also changed the name of the project to PiFarm so it sounds like a marketable product that has something to do with automation because it implies that a Raspberry Pi is involved. After porject lecture on 'Choosing the right database', we decided as a group that we would use Firebase as we require data to be managed in real time. Plus Zane and I had some prior experience with it so this was good.

Week 3: 07/03/2020

Milestone 1 Completed

Milestone 1 of the Project was completed and submitted which is the Project Scope Document. We decided that try to purchase the parts as my credit card is from a different bank than Zane's. We are currently waiting for clearance from the bank before we can make the purchase.

Week 4: 14/03/2020

Slow week

Research continued on other hydroponic/pi implementations

Week 5: 21/03/2020

Parts Ordered

Bank Clearance was received and the parts were ordered. Research to decide whether there is an available API we can use or if we would have to build one.

Week 6: 28/03/2020

Some parts arrived

Confirmation from Amazon that all parts have been shipped and all the things we ordered for the project arrived except the turbidity and pH sensors. No project meeting this week to carnival.

Week 7: 6/03/2020

Slow Week

Not much happened as I had 3 deadlines for my other courses to meet in this week. Had some discussions concerning building the web app with Gatsby js.

Week 8: 13/03/2020

University closed due to Corona Virus

Collected the Raspberry Pi 3 from Project Supervisor Inzamam Rahaman. Also switched frameworks for building the web app from Gatsby to React. On the 13th March, the Prime Minister of Trinidad and Tobago has closed schools and universities effective from the 14th March to the 20th of April due to the Corona Virus pandemic reaching the shores of our country.

Week 9: 20/03/2020

First week of online classes

DCIT responded very effectively and quickly as online classes commenced this week. The last two sensors arrived so coding on the Raspberry Pi can begin. Mid-course Presentation was originally due on the 17th due to circumstances, the deadline has been change to the 29th. Setup of the Raspberry Pi was done which included flashing the OS onto the new SD Card.

Week 10: 27/003/2020

Learning to Code for the Arduino Uno

Learnt about coding for the Arduino and that the mircocontroller does not stop executing whatever code is uploaded to it. For now, I connected the Arduino to my personal laptop and uploaded the code. No interaction with the Pi and Arduino yet. Using sample code from DFRobot website (the manufacturer), used the Arduino to take readings from the turbidity sensor. The following things were accomplished:
1) Used linear equation from DFRobot website for the turbidity sensor to convert the analog voltage read to NTU
2) Had to play with the trimmer on the sensor in order to get a stable reading from the sensor when it is inside the plastic coating and one high enough to be read as high through the analog input pins on the Arduino uno
See: https://www.teachmemicro.com/arduino-turbidity-sensor/
3) Adjusted linear equation to show the relationship between NTU and Voltage to reflect the reality of the sensor
See: https://wiki.dfrobot.com/Turbidity_sensor_SKU__SEN0189

Planning and filming for the mid-course video presentation was done. Could not show much implementation as all the sensors only arrived last week.
