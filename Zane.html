Throughout development of the Gatsby site, a number of plugins were added to the application to add functionality and styling. However, problems arose with setting up Firebase with Gatsby. From the getgo on the plugin documentation for Firebase, things didn’t exactly make much sense. The official documentation specified that the plugins would facilitate database access in the application, however, the database was not the only tool we needed to incorporate from Firebase to completely build the web app. We also needed authentication services to maintain user authentication tokens and possibly cloud functions to enable admin privileges on the application if it was required. Since the real time functionality Firebase offers was desired, and given my previous, pent up frustrations while working with the starter sites, we decided to abandon the Gatsby site and focus solely on a React implementation with firebase.Since Gatsby was already built with React anyways, this change was nothing difficult to carry out. In some instances, whole jsx files were copied to the new project.
